## VARIANT A — Web Scraping / Data Pipelines

**Title:**
Web Scraping & Data Pipelines in Python — Clean, Deduped, Repeatable

**Overview:**

I turn messy web sources into clean, deduplicated datasets you can actually use
— and I build it so it survives real conditions: pagination, rate limits, flaky
pages, and site changes.

My scraper engine is config-driven: the extraction logic lives in a config file,
so pointing it at a new site is a small change, not a from-scratch rebuild. It
dedupes on the fields you choose, respects robots.txt, retries with backoff, and
outputs to CSV or straight into Google Sheets. There's a working demo in my
portfolio that scrapes a live site end-to-end and produces the CSV.

Tell me the source and the fields you want. I'll hand you the data, the tool, or
both — and can run it on a schedule if you need it refreshed.

Working demo + source: https://github.com/lucamarraleexfreight-netizen/freelance-ops

## VARIANT B — Workflow Automation

**Title:**
Workflow Automation — Intake → Enrich → Draft → Human Approval (Python / n8n / Make)

**Overview:**

I automate the repetitive middle of your workflow and leave the judgment calls to
you. My reference build takes a lead list, enriches each row from public info
(site details, likely email, socials), drafts a personalized outreach email, and
drops every draft into an approval queue. Nothing is sent automatically — you
approve, then it's ready to go.

That human-approval gate is deliberate: auto-sending is how people get
blocklisted and how "automation" earns a bad name. The value is that the machine
does the grunt work and you sign off in seconds.

I work in plain Python (easy to host and own) or in n8n / Make when you want a
visual flow your team can edit. There's a runnable demo in my portfolio.

Working demo + source: https://github.com/lucamarraleexfreight-netizen/freelance-ops

**Skills:** Workflow Automation, Python, n8n, Make.com, Zapier, API Integration,
Email Automation, Data Enrichment, SQLite, Lead Generation, Process Automation,
CRM Integration.

## VARIANT C — AI Integration / Chatbots (RAG, Doc-Q&A)

**Title:**
AI Integration — RAG & Doc-Q&A Bots That Cite Their Sources

**Overview:**

I build "chat with your documents" bots that answer from *your* content and cite

portfolio you can look at.

If you have an internal knowledge base, support docs, or a contract/policy pile
