# Proposal Template

Short wins. The proposals that land are 4–6 sentences: prove you understood the
problem, show you've already built the thing, give a concrete plan + price, and
ask for one clear next step. No "Dear Hiring Manager," no life story.

## Variables

- `{client_name}` — their name if you have it, else drop the greeting.
- `{their_problem}` — their problem in their words, sharpened. Shows you read it.
- `{relevant_demo}` — the matching demo + its one-line proof (see below).
- `{price}` — fixed price for the scoped deliverable.
- `{turnaround}` — realistic delivery window.

## Demo references (use the one that matches the gig)

- **Scraper:** "a config-driven scraper I built that handles pagination,
  dedupe, rate-limiting and retries — here's it running on a live site with the
  CSV it produced"
- **Outreach:** "an intake→enrich→draft→approve pipeline I built where every
  email waits for one-click human approval before sending"
- **Doc-Q&A:** "a RAG doc-Q&A bot I built that answers from your documents and
  cites the source for every claim, in a clean web UI"

---

## Template

> Hi {client_name} — {their_problem_restated_sharply}.
>
> I've already built this: {relevant_demo}. ({GITHUB_URL})
>
> For you I'd {concrete_plan_in_one_or_two_sentences}. You'd get
> {deliverable}, done in {turnaround}, for a fixed **${price}**.
>
> {single_question_CTA}

---

## Fill guide

- **Hook:** mirror their problem back, one notch sharper than they wrote it.
- **Proof:** link the demo. This is the whole reason you win with no reviews.
- **Plan:** name the actual steps/output, not "I will help you with your
  project."
- **Price:** one fixed number for a scoped deliverable. Fixed-price + proof beats
  hourly for a no-review freelancer.
- **CTA:** exactly one question. "Want me to start with X?" not three options.
